기능 및 API 정리 (브레인스토밍)
김성보: 일단 가장 핵심 기능은 세 가지로 가죠.

성장 앨범: 사진/동영상 기록

성장 데이터: 키/몸무게 기록 및 그래프

일정 관리: 예방접종이나 병원 일정 알림

이현규: 앨범에는 카테고리를 넣으면 어떨까요? 그냥 사진만 저장하는 게 아니라 '웃음', '식사', '수면' 등으로 분류해서 나중에 찾아보기 쉽게. 발표할 때 사용자 편의성으로 어필하기 좋을 것 같아요.

허윤: (API 파일을 보며) 앨범 기능 구현하려면 cordova-plugin-camera랑 cordova-plugin-file는 필수겠네요. 사진 데이터는 Base64로 저장하는 것보다 파일 시스템에 저장하고 convertFileSrc로 불러오는 게 용량 면에서 좋을 거 같아요.

김성보: 성장 데이터 그래프는 Chart.js가 제일 깔끔할 것 같아요. 키랑 몸무게 이중 축 차트로 구현하면 발표 때 괜찮을꺼 같아요.

공민승: 그럼 데이터 저장 방식은 뭘로 할까요?

허윤: 일단은 브라우저 내장인 localStorage (Web Storage API)를 써서 빠르게 MVP를 만들고, 데이터가 너무 많아지면 나중에 SQLite로 바꾸는 방안으로 기재해두죠.

이현규: 일정 관리는 알림이 중요하죠.

허윤: cordova-plugin-local-notification이나 Web Notifications API를 쓰면 돼요. 알림 ON/OFF 기능도 넣는게 좋을거 같아요.

공민승: 그럼 세부 기능이랑 api는 정리 됬으니 각자 해보시고 내일은 어디까지 했는지 얘기 하는걸로 하시죠

🔨 개발 1차 점검 회의
김성보 (앱 제작/팀장): 제가 맡은 UI랑 핵심 구조는 index.html 파일 기반으로 jQuery Mobile을 써서 페이지 이동이랑 하단 탭 바까지 기본적인 틀은 잡았어요. 현규씨, UI 디자인은 깔끔하게 잘 나온 것 같아요?

이현규 (PPT/발표): 네, jQuery Mobile 테마 색상을 따뜻한 톤으로 잘 바꿔서 아기 앱 분위기에 딱 맞아요. 이제 이 틀에 기능만 잘 붙으면 발표 자료 만들기가 수월할 것 같아요.

허윤 (코드/플러그인): 성보씨, 저도 제가 맡은 플러그인 설정이랑 권한 부분은 거의 끝냈어요.

package.json이랑 config.xml에 cordova-plugin-camera, cordova-plugin-file, cordova-plugin-ionic-webview 다 추가하고 설정 완료했어요.

특히, 안드로이드 권한은 cordova-plugin-android-permissions를 써서 앱이 실행될 때 동적으로 권한 요청하도록 처리했어요. 그래야 안드로이드 6.0 이상에서 문제 없을거 같아요.

김성보: 그럼 이제 기능별로 들어가 보죠.

📸 성장 앨범 기능 구현 이슈
김성보: 제가 지금 앨범 기능에서 사진 찍고 저장하는 로직을 붙이고 있는데, 찍은 사진이 앱을 껐다 켜면 사라지는 문제가 있었어요.

허윤:  임시 URI 문제인거같아요. cordova-plugin-camera로 찍은 사진 경로는 임시 파일 시스템에 저장돼요. 그걸 영구 저장하려면 cordova-plugin-file의 copyToDataDirectory 같은 함수를 써서 앱의 영구적인 저장 공간으로 옮겨야 해요. 제가 readMe.md에 이 로직 설명 넣어뒀으니까 참고해주세요.

김성보:  파일 복사 로직을 추가해야 하는군요. 그리고 복사한 파일 경로는 웹뷰에서 바로 못 읽으니까, cordova-plugin-ionic-webview의 convertFileSrc()로 변환해서 써야 되는 거죠?

허윤:  그 변환된 URI를 localStorage에 저장하는 로직을 BabyApi 객체 안에 넣으면 돼요.

📊 성장 데이터 및 기타 기능 점검
허윤: 성보씨, 제가 그래프 시각화 쪽은 차트 샘플 데이터 넣어서 미리 봤는데, Chart.js 정말 잘 작동해요. 키랑 몸무게 이중 축 차트가 기대했던 대로 깔끔하게 나와요.

김성보:  그럼 제가 지금 데이터 입력 부분이랑 Chart.js 연동 로직만 완성하면 될 것 같아요. 입력된 데이터는 전부 localStorage에 JSON 형태로 저장할게요.

공민승 (회의록): 그럼 저희가 기획했던 대로, 데이터 입력 시 Web Storage API (localStorage)를 사용하고, 추후에 SQLite 전환 가능성을 남겨두는 거죠? 제가 회의록에 이 내용을 기록해두겠습니다.

허윤: 일정 관리의 알림 기능은 제가 Web Notifications API 기반으로 알림 요청하는 함수를 미리 만들어둘게요. 일단 이걸로 구현하고, 네이티브 알림은 마지막에 붙여보죠.

김성보:  그럼 다음 회의까지 각자 맡은 기능의 데이터 저장 및 불러오기 로직을 완성하는 것을 목표로 하시죠.
